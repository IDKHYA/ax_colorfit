# ColorFit UI 전면 이식 설계

작성일: 2026-07-10.

## 상태

사용자가 제공한 `colorfit-ui-v6.zip`과 V3~V6 수정 대화를 승인된 시각 기준으로 사용한다. 실제 구현 대상은 `v2/src`이며, `V2_UI`의 HTML과 코드 복사본은 비교 자료다.

## 목표

기존 퍼스널컬러 진단, 의류 분석, 옷장, 추천, 저장 코디, 데일리룩 로직을 유지하면서 한 제품으로 보이는 ColorFit UI를 만든다. PC와 모바일에서 같은 정보 우선순위를 유지하되 모바일은 촬영, 하단 시트, 캔버스 조작을 우선한다.

## 시각 체계

1. 기본 모드.
   홈, 옷장, 카탈로그, 옷 추가, 추천, 보관함, 데일리룩, 설정에 적용한다. 웜 그레이 배경, 높은 불투명도, 얕은 그림자, 네이비 단색 액션을 사용한다.
2. 크로마틱 모드.
   퍼컬 촬영, 설문, 결과에만 적용한다. 시즌 팔레트가 배경과 결과 히어로에 반영되고 24색 스펙트럼이 중심 정보가 된다.
3. 포커스 글라스 모드.
   색상 분석 팝업에만 적용한다. 선택 색상 기반 오브, LCh 좌표, 적합도, 최근접 팔레트, 대표·보조색 전환을 한 레이어에 모은다.

## 화면 구조

- 홈은 원본의 퍼컬 진단, 옷장 만들기, 옷 추가, 데일리룩 네 진입점을 유지한다. 날씨, 옷장 수, 최근 룩은 보조 정보다.
- 촬영은 실제 비디오를 큰 무대에 두고 얼굴·흰 종이 가이드와 상태만 글라스로 띄운다.
- 설문은 한 문항과 색상 스와치를 중심으로 표시하고 모바일에서는 선택지를 한 열로 쌓는다.
- 결과는 실제 Top1 시즌의 24색 팔레트, 4축, 관찰 색, 스펙트럼 위치, 12계절 비교, 옷장 활용색을 보여준다. 비교 선택은 화면 미리보기만 바꾸며 저장 결과는 바꾸지 않는다.
- 옷장 목록과 상세는 준비도 점수를 제거하고 개수, 최근 항목, 카테고리 수, 대표색을 보여준다.
- 옷 추가는 입력 이미지와 분석 요약을 먼저 보여준다. 상세 폼은 `정보 수정` 시트에서만 열린다.
- 추천은 1순위 조합을 가장 크게, 나머지 조합을 압축 카드로 보여준다. 점수는 퍼컬·날씨·조화·안정 네 축만 노출한다.
- 보관함은 폴더, 즐겨찾기, 검색, 정렬, 편집 진입으로 구성한다.
- 데일리룩은 캔버스가 중심이며 PC는 자산·캔버스·레이어 3열, 모바일은 캔버스·하단 도구 순서다.

## 상태와 저장

- 기존 `App.tsx`와 훅의 상태 소유권을 유지한다.
- 폴더 목록과 저장 룩의 `folderId`, `isFavorite`만 저장 계약에 추가한다.
- 기존 저장 데이터는 필드가 없어도 기본 폴더로 정규화한다.
- 퍼컬 시즌 비교, 색상 팝업 탭, 옷 추가 수정 시트 열림 상태는 화면 로컬 상태다.

## 접근성과 반응형

- 모든 아이콘 단독 버튼에 `aria-label`을 둔다.
- 모바일 고정 헤더와 하단 내비게이션을 고려해 안전 영역 패딩을 유지한다.
- 팝업과 수정 시트는 Escape, 배경 클릭, 닫기 버튼으로 닫힌다.
- 390px에서 수평 스크롤은 시즌·폴더 목록처럼 의도된 영역에만 허용한다.
- 드래그 중 React 상태 갱신은 `requestAnimationFrame`당 한 번으로 제한한다.

## 검증

- UI 계약 테스트로 화면별 핵심 클래스와 제거 대상 문구를 고정한다.
- 저장 폴더 정규화, 생성, 이동, 즐겨찾기를 테스트한다.
- 전체 타입 검사, 테스트, 빌드를 실행한다.
- 데스크톱과 모바일에서 핵심 화면을 순회하고 콘솔 오류와 오버플로를 확인한다.
