# ColorFit V5 Exact 통합 설계

## 목표

colorfit-react-v5-exact의 화면 구조와 Calm, Chromatic, Liquid Glass 규칙을 실제 v2 애플리케이션에 이식한다. 시각적 결과는 V5 Exact를 따르되, 퍼컬 진단, 이미지 분석, 옷장, 추천, 저장, PWA 로직은 현재 v2 구현을 유지한다.

## 기준 우선순위

1. 시각 구조와 화면 계층은 C:/Users/dheod/Downloads/colorfit-react-v5-exact/src/ColorFitPrototype.jsx와 src/styles.css를 기준으로 한다.
2. 데이터 소유권, 라우팅, 서비스 호출, 저장 방식은 v2/src를 기준으로 한다.
3. 폰트는 현재 v2의 Pretendard Variable 번들과 설정을 유지한다.
4. 참조 프로토타입의 하드코딩 데이터와 imperative controller는 이식하지 않는다.
5. 옷장 준비도, 옷장 건강도, 추천 준비 점수 개념은 UI와 상태 계약에서 제거한다.

## 화면 설계

### 홈

V5 Exact의 home-layout을 그대로 사용한다. 왼쪽에는 오늘 추천 CTA와 퍼컬, 등록 의류, 추천 조합 수를 배치하고, 오른쪽에는 날씨와 옷 추가, 퍼컬 결과 진입을 둔다. 아래에는 실제 옷장 3개와 최근 저장 룩을 표시한다. 표시 숫자와 이미지는 모두 현재 훅 데이터에서 계산한다.

### 퍼컬, 옷장, 옷 추가, 추천

현재 적용된 V5/V6 계열 React 구조를 V5 Exact의 클래스와 정보 계층에 맞춰 점검한다. 퍼컬 3단계와 색상 분석 팝업은 Chromatic/Liquid Glass를 유지하고, 일반 화면은 불투명도가 높은 Calm 표면을 사용한다. 옷 추가는 분석 우선, 정보 수정 시트 노출 원칙을 유지한다.

### 룩 보관함

V5 Exact의 vault-head, folder-row, saved-grid를 사용한다. 폴더는 장식이 아니라 실제 저장 계약이다. 전체, 즐겨찾기, 기본 폴더, 사용자 폴더 필터를 제공하고 룩을 폴더로 이동할 수 있어야 한다. 사용자 폴더는 생성, 이름 변경, 삭제가 가능하며 삭제 시 내부 룩은 기본 데일리 폴더로 이동한다.

### 데일리룩 편집

V5 Exact의 정적 데일리 편집기로 교체하지 않는다. 현재 TryOn.tsx의 누끼 처리, 레이어 캔버스, 드래그, 텍스트, 이미지 확정 로직을 유지한다. 시각 표면과 3열 및 모바일 순서만 공통 ColorFit 디자인에 맞춘다.

## 상태와 데이터 흐름

- App.tsx가 화면 전환과 도메인 훅 연결을 계속 소유한다.
- useWardrobes는 옷장과 의류만 관리하며 준비도 파생값을 더 이상 노출하지 않는다.
- useSavedOutfits가 저장 룩, 폴더, 즐겨찾기, 활성 편집 룩을 관리한다.
- 저장 룩과 폴더는 기존 persistence adapter를 통해 같은 저장 경로를 사용한다.
- 홈은 쓰기 로직을 갖지 않고 전달받은 실데이터를 시각화한다.

## 반응형 원칙

- 데스크톱은 V5 Exact의 사이드바, 상단 바, 2열 홈, 4열 옷장, 3열 데일리 편집 구조를 따른다.
- 모바일은 상단 헤더와 하단 내비게이션을 사용한다.
- 폴더 행은 가로 스크롤, 저장 룩은 1열, 데일리 캔버스는 자산 패널 다음에 배치한다.
- 문서 전체 가로 오버플로는 허용하지 않는다.

## 검증 기준

- 홈의 V5 Exact 핵심 클래스와 기능 진입이 유지된다.
- 소스와 화면에 준비도 개념이 남지 않는다.
- 폴더 생성, 이름 변경, 삭제, 룩 이동, 즐겨찾기 동작이 테스트된다.
- npm run lint, 전체 npm test, npm run build가 통과한다.
- 1280px 데스크톱과 390px 모바일에서 홈, 옷장, 옷 추가, 추천, 보관함, 편집기, 설정을 브라우저로 확인한다.
